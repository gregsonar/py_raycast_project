**Уточнение по №7:** `hit_info` присваивается только при `x == VIRT_WIDTH // 2`,
а для центрального столбца отклонение луча от направления взгляда равно нулю,
поэтому поправка fish-eye (`cos(0) = 1`) вырождается и ошибка сейчас не
проявляется. Баг реален, но «спящий»: код сломается при смещении точки
сэмплинга от центра. Тот же корневой изъян есть в вычислении `wall_hit`
(текстурная координата стены) — там скорректированный `dist` умножается на
вектор луча для **каждого** столбца; при FOV = 0.6 рад ошибка у края ~4%.
Исправляем оба места.
